# LUMIÈRE 商城 · 小组开发规范

> 本文档基于当前项目已落地的实现标准编写，所有成员在开发、联调、提交代码前请对照执行，保持前后台风格与行为一致。

---

## 1. 项目定位

- **项目名称**：LUMIÈRE 极简生活美学商城
- **形态**：单仓库、双端应用
  - **前台商城**（`/mall` 模块）：移动端 H5 体验，Ant Design Mobile + Tailwind
  - **后台管理**（`/admin` 模块）：PC 端，Ant Design + Tailwind
- **数据层**：开发环境使用 Mock.js 拦截 Axios，数据持久化在 `localStorage`，实现前后台联动

---

## 2. 技术栈（固定版本方向）

| 类别 | 选型 |
|------|------|
| 框架 | React 19 + Vite 6 |
| 路由 | React Router 7 |
| 样式 | Tailwind CSS v4 + 自定义 `@theme` 色板 |
| 前台 UI | antd-mobile 5 |
| 后台 UI | antd 5 |
| 状态 | Zustand（含 persist） |
| 请求 | Axios |
| Mock | Mock.js（仅 `import.meta.env.DEV` 启用） |

**路径别名**：统一使用 `@` 指向 `src/`，禁止相对路径跨多层 `../../`。

**入口约定**：
- `main.jsx` 中引入 `@/mock`、`@/index.css`
- React 19 需保留 `unstableSetRender` 以兼容 antd-mobile 命令式组件

## 3. 目录结构

```
src/
├── mall/                 # 前台商城（成员按模块认领）
│   ├── pages/            # 页面
│   ├── components/       # 业务组件
│   ├── hooks/            # 前台 hooks
│   ├── store/            # Zustand 状态
│   ├── constants/        # 常量（auth、validation、order 等）
│   └── utils/            # 前台工具（如 toast）
├── admin/                # 后台管理
│   ├── pages/
│   ├── components/
│   ├── hooks/
│   ├── store/
│   └── utils/
├── components/           # 跨端公共组件（暂无则保留 .gitkeep）
├── hooks/
├── store/
├── utils/                # 公共工具：api.js、request.js、auth.js
├── mock/                 # Mock 数据中心（统一维护 API）
├── router/               # 路由配置
├── App.jsx               # 根布局：按 /admin 切换 ConfigProvider
└── main.jsx
```

**约定**：
- 新页面放在对应端的 `pages/` 下，每个页面一个文件夹 + `index.jsx`
- 可复用 UI 放 `components/`，命名 PascalCase 文件夹（如 `GlassSearchNav/index.jsx`）
- 业务常量放 `constants/`，不要硬编码在组件里
- API 调用统一走 `@/utils/api.js`，禁止在页面里直接写 `axios.get('/api/...')`

## 4. 视觉与设计规范

### 4.1 设计理念

- **风格**：极简、温暖、现代
- **布局**：前台最大宽度 `max-w-lg` 居中，模拟移动端；后台全宽 PC 布局
- **圆角**：卡片 `rounded-2xl` / `rounded-3xl`，按钮 `rounded-full` 或 `shape="rounded"`
- **质感**：前台适当使用毛玻璃（`backdrop-blur-xl`、`bg-white/90`）

### 4.2 主体色调（必须遵守）

色值定义在 `src/index.css` 的 `@theme` 中，**禁止随意引入新主色**。

| 角色 | 色板 | 典型用法 |
|------|------|----------|
| **页面背景** | `cream-50` `#fdfcf9` | `body`、商城页底色 |
| **卡片/区块背景** | `cream-100` ~ `cream-200` | 分隔、次要背景 |
| **主色（品牌色）** | `olive-600` `#4a6340` | 主按钮、Tab 选中、强调文字 |
| **主色深/浅** | `olive-700` ~ `olive-800` / `olive-50` ~ `olive-200` | 标题、hover、浅色标签背景 |
| **辅助强调色** | `sea-*` 深海蓝 | 订单「待发货」等次要状态 |
| **中性文字** | `stone-400` ~ `stone-800` | 正文、说明、占位 |

**Ant Design 主题 Token**（后台 `App.jsx`）：

```js
colorPrimary: '#4a6340'
colorBgContainer: '#fdfcf9'
colorBgLayout: '#f8f6f0'
borderRadius: 8
fontFamily: '"Inter", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif'
```

**前台主按钮**：antd-mobile `Button` 需设置 `style={{ '--background': '#4a6340' }}`。

### 4.3 字体

统一使用：`Inter`、`PingFang SC`、`Microsoft YaHei`、`system-ui`。

### 4.4 骨架屏

- 加载态使用 `@/mall/components/PageSkeleton` 中已有骨架（`AuthPageSkeleton`、`HomePageSkeleton`、`ListPageSkeleton`）
- 骨架块使用 `cream-200` 渐变脉冲，与主色系统一致
- 鉴权恢复期间（`useAuthHydration` 未 `hydrated`）必须显示骨架，禁止白屏闪烁

## 5. 前台商城规范

### 5.1 登录门禁（强制）

- **打开网站必须先登录**：除 `/login`、`/register` 外，所有商城路由由 `MallAuthGuard` 保护
- 未登录访问任意商城路径 → 跳转 `/login`，并通过 `state.from` 记录来源，登录成功后回跳
- 已登录用户访问 `/login`、`/register` → `MallGuestGuard` 自动跳转首页

### 5.2 登录态存储

| 端 | Storage Key | 存储位置 |
|----|-------------|----------|
| 前台 | `mall-token` | `sessionStorage`（关闭浏览器需重新登录） |
| 前台用户态 | `mall-user-storage` | `sessionStorage`（Zustand persist） |
| 后台 | `admin-token` | `localStorage` |
| 后台用户态 | `admin-auth-storage` | `localStorage` |

**前后台 Token 必须分离**，禁止共用同一个 key。

### 5.3 注册 / 登录表单

| 字段 | 规则 |
|------|------|
| 用户名 | 3–20 位，仅字母、数字、下划线；不得与已有用户重复 |
| 密码 | 至少 6 位，**必须同时包含字母和数字** |
| 确认密码 | 与密码一致 |
| 注册表单 | **禁止** `initialValues` 自动填充；`autoComplete="off"` / `new-password` |
| 校验常量 | 统一引用 `@/mall/constants/validation.js` |

登录页可保留演示账号提示（`user / 123456`），仅供登录 Tab 使用。

### 5.4 页面容器

- 带底部 Tab 的页面使用 `MallPageShell` 包裹
- 底部 Tab：`首页 / 购物车 / 我的订单`（`MallTabBar`）
- 操作反馈统一使用 `mallToast`（`success` / `fail` / `info`），**禁止**直接使用 antd-mobile 命令式 `Toast`（React 19 不兼容）

### 5.5 订单状态

定义在 `@/mall/constants/order.js`：

| status | 含义 |
|--------|------|
| `0` | 待支付 |
| `1` | 待发货 |
| `2` | 已完成 |

状态标签颜色：`待支付` → amber，`待发货` → sea，`已完成` → olive。

## 6. 后台管理规范

### 6.1 路由与权限

- 未登录 → `/admin/login`
- 登录后由 `AuthGuard` + `PermissionGuard` 控制模块访问
- 无权限 → `/admin/forbidden`

| 角色 | 账号 | 密码 | 权限 |
|------|------|------|------|
| 超级管理员 | `admin` | `admin123` | 全部模块 |
| 商品测试 | `test` | `123456` | 仅商品 |
| 运营 | `operator` | `123456` | 仅订单 |
| 前台用户 | `user` | `123456` | 不可登录后台 |

**角色隔离**：
- 前台 `login` 仅允许 `role === 'user'`
- 后台 `login` 拒绝 `role === 'user'`

### 6.2 后台 UI

- 使用 Ant Design 表格、表单、Modal 等 PC 组件
- 主题色与前台一致（`#4a6340`）
- 占位模块使用 `PlaceholderPage`，标题通过 props 传入

## 7. API 与 Mock 规范

### 7.1 响应格式（统一）

```json
{
  "code": 0,
  "data": { },
  "message": "success"
}
```

- 成功：`code === 0` 或 `code === 200`
- 失败：`code !== 0`，`message` 为可展示的错误文案
- 鉴权失败：`code === 401`

### 7.2 请求规范

- `baseURL`: `/api`
- 鉴权头：`Authorization: Bearer <token>`
- 前台/后台 Token 由 `request.js` 拦截器按当前路径自动选择
- 401 时：清除前台 session 并 `logout`（排除 `/login`、`/register`）

### 7.3 Mock 维护

- 所有接口在 `src/mock/index.js` 统一注册
- 业务数据 key：`mock_products`、`mock_categories`、`mock_cart`、`mock_orders`、`mock_users`、`mock_tokens`
- 修改种子数据或新增接口时，需保证**前台与后台读写同一份持久化数据**
- Mock 仅在开发环境生效，生产构建不包含 Mock 逻辑

### 7.4 新增 API 流程

1. 在 `src/utils/api.js` 添加方法
2. 在 `src/mock/index.js` 添加对应 `Mock.mock(...)`  handler
3. 页面/Store 通过 api 方法调用，不直连 Mock

## 8. 状态管理（Zustand）

| Store | 文件 | persist key | 职责 |
|-------|------|-------------|------|
| 前台用户 | `useMallUserStore` | `mall-user-storage` | 登录、注册、登出 |
| 购物车 | `useCartStore` | `mall-cart-storage` | 增删改查、选中态 |
| 订单 | `useOrderStore` | `mall-order-storage` | 订单列表与详情 |
| Toast | `useToastStore` | 无 | 全局提示 |
| 后台 | `useAdminStore` | `admin-auth-storage` | 登录、权限判断 |

**约定**：
- Store 文件命名：`useXxxStore.js`
- 异步 action（login、register 等）在 Store 内完成，页面只调用并处理 UI 反馈
- 需要等待 persist 恢复的场景，使用 `useAuthHydration()`，不要直接读 Store 判断首屏登录态

## 10. 代码风格与协作

### 10.1 命名

- 组件：PascalCase（`ProductCard`）
- 文件/文件夹：组件目录 PascalCase，工具/Store camelCase
- 常量：UPPER_SNAKE_CASE（`ORDER_STATUS`、`MALL_TOKEN_KEY`）
- CSS：优先 Tailwind 工具类，不写零散全局 CSS（特殊动画除外）

### 10.2 组件编写

- 函数组件 + Hooks，默认导出页面/组件
- 业务逻辑能抽 Hook 则抽 Hook（如 `useAuthHydration`）
- 避免在 JSX 里写大段内联样式，品牌色用 Tailwind 类或 CSS 变量

### 10.3 Git 协作建议

- 按模块认领目录（如 A 负责 `mall/pages/Orders`，B 负责 `admin/pages/Product`）
- 共用文件（`router`、`mock`、`api.js`、`request.js`）修改前先沟通
- 提交前执行 `npm run build` 确保无编译错误
- 提交信息简洁说明「为什么改」，中英文均可

### 10.4 禁止事项

- ❌ 前台绕过 `MallAuthGuard` 新增公开页面
- ❌ 混用前台/后台 Token
- ❌ 在组件内硬编码 `#4a6340` 以外的随机主色
- ❌ 使用 antd-mobile 命令式 Toast / Dialog（除非已做 React 19 适配）
- ❌ 在 `pages/` 里直接写 fetch/axios
- ❌ 修改 Mock 数据结构却不更新 `api.js` 与对应 Store
